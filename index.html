1) ¿Qué versiones de JavaScript hay en el mercado?

Javascript es definido y estandarizado por una organización internacional llamada Ecma Iternational. Es el organismo actualmente responsable de la definición de la especificación oficial JavaScript, a la que se denomina ECMAScript o Standard ECMA-262. Esta sirve para que las empresas o entidades que desarrollan navegadores web reconozcan JavaScript y pueda ser usado por los programadores web, y así todos usen el mismo lenguaje y obtengan los mismos resultados.

Aunque todos los navegadores usan JavaScript a partir de la especificación ECMAScript, hay matices, diferentes particularidades y resultados entre navegadores.

La Mozilla Foundation, creadora del navegador Mozilla Firefox, mantiene una numeración de versiones (1.7, 1.8, 1.8.1, …1.8.5, 1.9, 1.15, 1.20 …) que indica la evolución del intérprete JavaScript dentro del navegador:

  • ES1: (Junio 1997) Primera versión estandarizada. Tiene muchas de las características que se usan hoy en día, como objetos y herencia prototipal. Procuraba definir un lenguaje apropiado para dotar de más dinamismo a la web.
  • ES2: (Junio 1998) Estandarizado, casi sin cambios.
  • ES3: (Diciembre 1999) Funcional en todos los navegadores (excepto las versiones anteriores de Internet Explorer). Agrega soporte para igualdad estricta, manejo de excepciones, etc.
  • ES5: (Diciembre 2009)
  • ES5.1: (Junio 2011)
  • ES6: (Junio 2015)Añadió una serie de características muy importantes demandadas cada vez más por los programadores, las cuales supusieron un cambio importante en el lenguaje. Son extensamente utilizadas por todos los frameworks actuales y han sido incorporadas a los entornos de javascript modernos, tanto en los navegadores como en node.js. A partir de este año las actualizaciones son anuales para evitar cambios tan bruscos.
  • ES7: (Junio 2016)Una actualización que añade pocos cambios comparada a su versión anterior.
  • ES8: (Junio 2017)Introduce algunos cambios mayores y unos menores pero que vienen a ser de mucha ayuda en el día a día. Permite dejar comas sueltas al declarar objetos, arrays y parámetros para funciones.
  • ES9: (Junio 2018)Incluye mejoras a las cosas que ya estaban.
  • ES10: (Junio 2019)
  • ES11: (Junio 2020)
  • ES12: (Junio 2021)

Desde ES1 a ES6 es lo que se conoce como código seguro ya que es compatible con todos los navegadores y motores que corran JavaScript al 100%.

2) ¿Cómo se integra JavaScript a HTML?

JavaScript es un lenguaje utilizado principalmente con el lenguaje HTML, es un lenguaje interpretado que se embebe en una página web HTML. Un lenguaje interpretado significa que a las instrucciones las analiza y procesa el navegador en el momento que deben ser ejecutadas. Hay varias formas de introducir JavaScript a HTML:

  • Agregar JavaScript directamente a un archivo HTML usando las etiquetas <script> </script>, envolviendo todo el código JS que se agrega:
    • Entre las etiquetas <head> </head>
    • Entre las etiquetas <body> </body>
  • Agregar código JavaScript a un archivo separado (archivo externo):
    • Entre las etiquetas <body> </body>
      Ejemplo: <script src="./script.js"></script>
      Esto apuntaría a un archivo llamado "script.js" que está en el mismo directorio o carpeta del archivo .html. También se puede enlazar a otras carpetas usando ".." en la ruta del archivo.
      Ejemplo: <script src="js/script.js"></script> Este ejemplo también apunta a un archivo llamado "script.js" pero en otra carpeta distinta a la del archivo .html.
    • Entre las etiquetas <head> </head> de la misma forma que en el body.

Variables

Las variables son depósitos donde se guarda información, un espacio de almacenamiento de reserva en memoria. Tienen un nombre y un tipo (valor entero, valor real, cadena de caracteres, valores lógicos). 4) ¿Cómo son todas las formas de declaración de funciones en JS?

Hay 4 formas de declarar variables:

  • usando var
  • usando let
  • usando const
  • sin usar nada

Ejemplos:

  • var marcaGaseosa = "Fanta";
  • let nombre = "Pepito";
  • const pi = 3.14;
  • x = 2;

Las variables siempre se declaran con var, let o const. la palabra var es usada en todos los códigos en todos los códigos JS desde 1995 a 2015. Las palabras let y const fueron añadidas en la versión de JS de 2015. Es por esto que si se quiere ejecutar el código en navegadores antiguos se debe usar var. Como regla general las variables siempre se declaran con const, pero si el valor de la variable puede cambiar entonces se usa let.

Var

Declara una variable, iniciandola opcionalmente en un valor, que podrá cambiar. Su scope es local.

Let

Declara una variable local en un bloque de ámbito, iniciandola opcionalmente en un valor. Su valor puede cambiar.

Const

Declara una variabled e solo lectura en un bloque de ámbito. No es posible cambiar su valor mediante la asignación.

5) ¿Cuáles son los operadores y conversión entre tipos?

Operador de asignación

El operador de asignacion es el símbolo de igualdad (=), y sirve para asignarle un valor a una variable.

Adición o suma

Se usa el símbolo "+"
Ejemplo: a + b

Multiplicación

Se usa el símbolo "*"
Ejemplo: a * b

Sustracción o resta

Se usa el símbolo "-"
Ejemplo: a - b

División

Se usa el símbolo "/"
Ejemplo: a / b

Resto o módulo

Se usa el símbolo "%"
Ejemplo: a % b

Exponenciación

Se usa el simbolo "**"
Ejemplo: a ** b -> Eleva "a" a la potencia "b" (ab). Equivale a Math.pow(a, b).

Operadores de comparación

Aquellos que se utilizan en el código (generalmente, en el interior de un if, aunque no es el único lugar donde se pueden usar) para realizar comprobaciones. Estas expresiones de comparación devuelven un booleano con un valor de true o false.

  • Igualdad Devuelve el valor true si ambos operandos son iguales (==)
  • Desigualdad Devuelve el valor true si ambos operandos NO son iguales (!=)
  • Estrictamente iguales Devuelve el valor true si ambos operandos son iguales y tienen el mismo tipo (===)
  • Estrictamente desiguales Devuelve el valor true si ambos operandos no son iguales y/o no son del mismo tipo.
  • Mayor que Devuelve el valor true si el operando de la izquierda es mayor al de la derecha(>)
  • Mayor o igual que Devuelve el valor true si el operando de la izquierda es mayor o igual al de la derecha (>=)
  • Menor que Devuelve el valor true si el operando de la izquierda es menor al de la derecha (<)
  • Menor o igual que Devuelve el valor true si el operando de la izquierda es menor o igual al de la derecha (<=)

Operadores aritméticos

  • incremento (++) Incrementa en una unidad al operando
    • Si se usa ++x devuelve el valor del operando despues de añadirle 1. Ejemplo: si x=3 y se usa ++x entonces devuelve 4.
    • Si se usa x++ devuelve el valor del operando antes de añadirle 1. Ejemplo: si x=3 y se usa x++ entonces devuelve 3 y después establece 4.
  • decremento (--) Resta una unidad al operando, y depende su posición respecto al operando tiene el mismo comportamiento que el incremento.
    • Si se usa --x y x=3 entonces x=2
    • Si se usa x-- y x=3 entonces devuelve 3 y luego establece que x=2

Operador de concatenación

Une dos valores de tipo String, devolviendo otro String correspondiente a la unión.
Ejemplo: console.log("soy" + "Zahira"); // Soy Zahira

Operadores binarios

Permiten realizar operaciones básicas trabajando a nivel binario, donde los operandos solo pueden tomar valores de 0 o 1.

  • AND

    Se usa el símbolo "&"
    Ejemplo:
    a & b -> Devuelve 1 si ambos operandos son 1.

  • OR

    Se usa el símbolo "|"
    Ejemplo:
    a | b -> Devuelve 1 si al menos un operando es 1.

  • XOR (OR exclusivo)

    Se usa el símbolo "^"
    Ejemplo:
    a ^ b -> Devuelve 1 si ambos operandos son diferentes.

Conversión de tipos de datos

  • ToString

    Se usa cuando se necesita la representación en forma de texto de un valor. Se puede también llamar a la función String(value) para convertir un valor a string.

    Por ejemplo: El boolean false se convertiría en "false", o null pasaría a ser "null"

  • ToNumber

    Ocurre automáticamente en funciones matemáticas y expresiones. Se puede usar la función Number(value) para convertir de forma explícita un valor a un número y es usualmente requerida cuando leemos un valor desde una fuente basada en texto, como los campos de texto en los formularios, pero que se espera que contengan un valor numérico.

    Si el string no es un número válido el resultado de la conversión da como resultado NaN (Not a Number). También se eliminan los espacios (incluyendo tabs, saltos de línea) al inicio y final del texto. Si el string esta vacío, el resultado será 0. Null y undefined se comportan de distinta manera; null se convierte en 0 y undefined pasa a ser NaN.

    Casi todas las opraaciones matemáticas convierten valores a números, pero hay una excpción en las sumas; si uno de los valores que se suma es un string, el otro valor también se convierte a string y luego los concatena o une. Por ejemplo:

    alert( 1 + '2' ); // Da como resultado '12'

    alert( '1' + 2 ); // Da como resultado '12'

    Esto ocurre solo si al menos uno de los argumentos es un string, en caso contrario los valores son convertidos a número.

  • ToBoolean

    Es la conversión más simple, y se usa en operaciones lógicas, test condicionales y similares. Puede realizarse de forma explícita llamando la función Boolean(value).

    Como reglas de conversión, los valores "vacíos" como 0, "null", "", undefined, y NaN son convertidos a false, y otros valores se convierten en true. Un string con valor "0" es tomado como true.
